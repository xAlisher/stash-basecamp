# Stash 0.1.0 — Decentralised Backup for Logos Basecamp

*2026-04-22*

Today I'm releasing **Stash 0.1.0**, a Logos Basecamp module that gives any
module a one-click path to back up its data to decentralised storage.

---

## The problem it solves

Logos Basecamp modules run in isolated sandboxes with no internet access from
QML. There's no built-in backup mechanism. Each module that wants to back up
data would have to wire up its own IPFS client, manage upload state, handle
pinning, and surface a UI — duplicating the same infrastructure across every
module.

Stash pulls all of that into a single reusable module. A module that wants
backup support needs exactly two C++ methods and a one-line QML component.

---

## What Stash does

**Three transport options:**

| Transport | How it works |
|-----------|-------------|
| **Logos Storage** | Uploads via the `storage_module` IPC — the native Logos P2P storage layer. No external accounts needed. |
| **Kubo** | Bundles a local Kubo (IPFS) node and pins to a configurable endpoint. |
| **Pinata** | Pins directly to Pinata v3 API using your API key. |

The active transport is set once in the Stash settings panel and applies to
all modules automatically. Switching transports takes effect on the next upload.

**Activity log** — every upload is recorded with timestamp, filename, and CID:

```
[15:16:49] storage_module ready
[15:16:53] notes: 2859e8fee220d620_2026-04-22_1516.imnotes
[15:16:53] Logos Storage: uploading 2859e8fee220d620_2026-04-22_1516.imnotes
[15:17:15] Logos Storage: 2859e8fee220d620_2026-04-22_1516.imnotes → zDvZRwzm...
```

## Integrating Stash into your module

Drop in two files and add one line of QML.

**C++ plugin** — two `Q_INVOKABLE` methods:

```cpp
// Return the file you want backed up
Q_INVOKABLE QString getFileForStash();
// → {"ok":true,"path":"/absolute/path/to/file"}

// Stash calls this with the CID when the upload settles
Q_INVOKABLE QString setBackupCid(const QString& cid, const QString& timestamp);
// → {"ok":true}
```

**QML** — copy `StashButton.qml` from the integration directory and drop it
into your plugin:

```qml
StashButton {
    moduleName: "your-module-name"
}
```

That's it. The button reads the active transport, exports the file from your
module, uploads it, waits for the CID, and calls `setBackupCid` on your
module when done — across all three transports.

See [`docs/stash-button-integration.md`](../docs/stash-button-integration.md)
for the full guide including the IPC flow, customisation properties, and
activity log format.

## How async delivery works

The Logos Storage transport is fully asynchronous — the upload takes ~20 seconds
and the CID arrives via a storage event. Getting this right required solving a
non-obvious IPC constraint.

The naive approach — have Stash call `setBackupCid` back on the originating
module when the CID arrives — consistently timed out. The root cause: after
module A calls Stash via IPC, Stash cannot call back into A while A is still
processing the response. `requestObject` and `invokeRemoteMethodAsync` both
time out in this context regardless of whether the client pointer is fresh.

The fix is architectural. Stash stores the upload result (`file`, `cid`, `ts`)
and exposes it via `getLatestLogosResult()`. The calling module polls this every
2 seconds and calls `setBackupCid` on itself when its file's CID appears. All
of this is handled inside `StashButton` — modules using the button don't need
to know about it.

This pattern — **store and poll instead of back-call** — is now documented in
[`basecamp-skills`](https://github.com/xAlisher/basecamp-skills) as
`ipc-back-call-use-poll-push` for future modules facing the same constraint.

## Install

Download both LGX files from the [0.1.0 release](https://github.com/xAlisher/stash-basecamp/releases/tag/v0.1.0).

```bash
LGPM=/path/to/lgpm
MDIR=~/.local/share/Logos/LogosBasecamp/modules

# Install core module
$LGPM --modules-dir $MDIR \
      --ui-plugins-dir ${MDIR%modules}ui-plugins \
      --allow-unsigned install --file logos-stash-module.lgx

# Install UI plugin
$LGPM --modules-dir $MDIR \
      --ui-plugins-dir ${MDIR%modules}ui-plugins \
      --allow-unsigned install --file logos-stash_ui-module.lgx
```

Restart Logos Basecamp. Stash appears in the sidebar.
